# Token-Spike Interface

Преобразование между языковыми токенами и спайковыми паттернами нейронной сети.

## Концепция

LLM (языковые модели) мыслят **токенами** - кусочками слов. Спайковые нейронные сети мыслят **импульсами во времени**. Этот модуль - транслятор между двумя мирами.

## Архитектура

```
Слово "Привет"
    ↓
Фонемы: ['п', 'р', 'и', 'в', 'е', 'т']
    ↓
Частоты: [40Hz, 35Hz, 42Hz, 38Hz, 40Hz, 37Hz]
    ↓
Спайковый паттерн: [(0ms, n10), (25ms, n10), (40ms, n26), ...]
    ↓
Нейронная сеть обрабатывает
    ↓
Резонансный анализ
    ↓
Декодированное слово
```

## Использование

### Энкодирование (слово → спайки)

```rust
use token_spike_interface::encoder::PhonemeEncoder;

let encoder = PhonemeEncoder::new();

// Одно слово
let pattern = encoder.encode("привет");
println!("Спайков: {}", pattern.spikes.len());
println!("Длительность: {}мс", pattern.duration);
println!("Частота: {}Hz", pattern.average_frequency());

// Предложение
let sentence_pattern = encoder.encode_sentence("привет мир");
```

### Декодирование (спайки → слово)

```rust
use token_spike_interface::decoder::SpikeDecoder;
use token_spike_interface::encoder::PhonemeEncoder;

let mut decoder = SpikeDecoder::new();

// Добавляем словарь
decoder.add_words(&["привет", "пока", "спасибо", "мир"]);

// Декодируем
let encoder = PhonemeEncoder::new();
let pattern = encoder.encode("привет");

let word = decoder.decode(&pattern);
assert_eq!(word, Some("привет".to_string()));

// С оценками всех кандидатов
let scores = decoder.decode_with_scores(&pattern);
for (word, score) in scores {
    println!("{}: {:.2}", word, score);
}
```

### Резонансный анализ

```rust
use token_spike_interface::resonance::ResonanceAnalyzer;

let analyzer = ResonanceAnalyzer::new();

let resonance = analyzer.calculate_resonance(&pattern1, &pattern2);
println!("Резонанс: {:.2}", resonance); // 0.0 - 1.0

// С учетом частоты
let resonance_freq = analyzer.calculate_resonance_with_frequency(&pattern1, &pattern2);

// Автоматическое выравнивание
let (best_resonance, shift) = analyzer.find_best_alignment(&pattern1, &pattern2);
println!("Лучший резонанс: {:.2} при сдвиге {}мс", best_resonance, shift);
```

## Как это работает

### 1. Фонемы → Частоты

Каждая фонема (звук) имеет характерную частоту и длительность:

| Фонема | Частота | Длительность | Тип |
|--------|---------|--------------|-----|
| 'а' | 42 Hz | 80 мс | Гласная |
| 'п' | 40 Hz | 40 мс | Взрывная |
| 'р' | 35 Hz | 50 мс | Сонорная |
| 'ш' | 52 Hz | 70 мс | Фрикативная |

### 2. Частоты → Спайки

Для каждой фонемы генерируются спайки с её частотой:

```
'п' (40 Hz, 40 мс):
  Интервал = 1000/40 = 25 мс
  Спайки: 0мс, 25мс
  
'р' (35 Hz, 50 мс):
  Интервал = 1000/35 = 28.6 мс
  Спайки: 40мс, 68.6мс
```

### 3. Резонансный анализ

Для декодирования используется **кросс-корреляция**:

```rust
for каждый_спайк in наблюдаемом_паттерне {
    найти_ближайший_спайк в ожидаемом_паттерне;
    если расстояние < 50мс {
        резонанс += exp(-расстояние / 50);
    }
}
```

Слово с максимальным резонансом (> 0.7) - это результат декодирования.

## Научная основа

### Временное кодирование

В реальном мозге информация кодируется **временем спайков**, а не только их частотой. Это называется **temporal coding**.

### Фазовая синхронизация

Нейроны, обрабатывающие связанную информацию, синхронизируют свои спайки. Модуль использует это для анализа резонанса.

### Гамма-осцилляции

Частоты 30-100 Hz (гамма-ритм) связаны с обработкой языка в мозге. Наши фонемные частоты находятся в этом диапазоне.

## Производительность

- **Энкодирование**: O(n) где n - длина слова
- **Декодирование**: O(v × m) где v - размер словаря, m - количество спайков
- **Резонанс**: O(n × m) где n, m - количество спайков в паттернах

Для словаря из 1000 слов декодирование занимает ~1-2мс.

## Ограничения

1. **Упрощенная фонетика**: Используются отдельные символы, а не настоящие фонемы
2. **Фиксированные частоты**: В реальности частоты варьируются
3. **Нет контекста**: Не учитывается семантика слов
4. **Только русский и английский**: Другие языки не поддерживаются
