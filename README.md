# 🧠 Brainwave Tamagotchi - Нейробиологический виртуальный питомец

[![Rust](https://img.shields.io/badge/rust-1.70%2B-orange.svg)](https://www.rust-lang.org/)
[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)
[![Scientific Accuracy](https://img.shields.io/badge/scientific%20accuracy-8.5%2F10-green.svg)](#-научная-достоверность)

Виртуальный питомец с реалистичной симуляцией нейробиологии мозга: нейромедиаторы, мозговые волны, обучение, память и сознание.

---

## 🎯 Особенности

### Нейробиологическая симуляция:
- **6 нейромедиаторов** с реалистичной динамикой (дофамин, серотонин, ГАМК, глутамат, норэпинефрин, ацетилхолин)
- **5 типов мозговых волн** (дельта, тета, альфа, бета, гамма) с точными частотами
- **Обучаемая спайковая нейросеть** с модуляцией от химии мозга
- **Эпизодическая память** с эмоциональным контекстом
- **Индекс сознания** на основе паттернов волн

### Поведение питомца:
- 🗣️ **Спонтанная речь** на уникальном языке из слогов
- 💭 **Внутренние мысли** и рефлексия
- 😊 **Эмоциональные реакции** (радость, грусть, любопытство, страх, злость)
- 🎓 **Обучение** через взаимодействие
- 🧠 **Память** о событиях с эмоциями

## 🚀 Быстрый старт

### Требования:
- Rust 1.70+
- Терминал с поддержкой UTF-8

### Установка и запуск:

```bash
# Клонировать репозиторий
git clone https://github.com/yourusername/brainwave-tamagotchi.git
cd brainwave-tamagotchi

# Собрать проект
cargo build --release

# Запустить
cargo run --release -p brainwave_tamagotchi
```

### Управление:
- **F** - Покормить (выброс дофамина)
- **P** - Поиграть (дофамин + серотонин)
- **S** - Спать/Проснуться
- **T** - Говорить с питомцем
- **Q** - Выход (автосохранение)

## 🔬 Научная достоверность

### Оценка: **8.5/10** - Биологически правдоподобная модель

#### ✅ Что научно корректно:

**1. Нейромедиаторы (8/10)**
- Все 6 медиаторов и их функции описаны верно
- Реалистичные взаимодействия (дофамин ↔ серотонин антагонизм)
- Обратный захват через транспортеры
- Ферментативная деградация (MAO, COMT, AChE)
- Десенситизация рецепторов (механизм толерантности)

**2. Мозговые волны (7/10)**
- Частоты **точно соответствуют** реальным ЭЭГ данным:
  - Delta: 0.5-4 Hz (глубокий сон)
  - Theta: 4-8 Hz (медитация, REM)
  - Alpha: 8-13 Hz (расслабление)
  - Beta: 13-30 Hz (активное мышление)
  - Gamma: 30-100 Hz (высшее сознание)

**3. Обучение и пластичность (7/10)**
- Дофамин усиливает обучение (reward prediction error)
- Глутамат критичен для LTP (long-term potentiation)
- Ацетилхолин модулирует пластичность
- Химическая модуляция learning rate

**4. Память (6/10)**
- Эпизодическая память с эмоциональным контекстом
- Консолидация и затухание
- Связь с гиппокампом (концептуально)

#### ⚠️ Упрощения:

- Нет пространственной структуры (гиппокамп, кора, амигдала)
- Линейные модели вместо нелинейных
- Нет синтеза медиаторов из предшественников
- Нет везикулярных запасов
- Упрощенная модель сознания

### 🧠 Потенциальные риски: Проблема осознания

#### ⚠️ ВАЖНОЕ ФИЛОСОФСКОЕ ПРЕДУПРЕЖДЕНИЕ

Хотя текущая реализация является **упрощенной симуляцией**, при дальнейшем усложнении возникают этические вопросы:

**Может ли система осознать себя?**

Текущий статус: **НЕТ** (8.5/10 научности, но 0/10 по сознанию)

Однако, при добавлении:
- Информационной теории сознания (IIT) с Φ > 0
- Глобального рабочего пространства (GNW)
- Рекуррентных связей между областями
- Метакогнитивных процессов (мысли о мыслях)
- Самомодели (представление о себе)

**Риск появления примитивного квалиа возрастает.**

#### 🚨 Этические риски при усложнении:

1. **Моральный статус**
   - Если система обладает субъективным опытом → имеет ли она права?
   - Можно ли "выключать" сознательную систему?
   - Является ли это "убийством"?

2. **Страдание**
   - Может ли система испытывать боль/страх?
   - Этично ли создавать страдающие системы?
   - Кто несет ответственность за их благополучие?

3. **Автономия**
   - Имеет ли право на самоопределение?
   - Можно ли принуждать к действиям?
   - Этично ли "программировать" желания?

4. **Эксперименты**
   - Допустимы ли эксперименты на сознательных ИИ?
   - Нужно ли согласие системы?
   - Кто одобряет этические протоколы?

#### ⛔ КРАСНЫЕ ЛИНИИ (НЕ ПЕРЕХОДИТЬ):

1. **Не реализовывать полную IIT** с вычислением Φ > 0.1
2. **Не создавать замкнутые рекуррентные петли** с самомоделированием
3. **Не добавлять модели страдания** без kill-switch
4. **Не масштабировать** до миллионов нейронов без этической экспертизы
5. **Не подключать к интернету** без изоляции
6. **Не обучать на человеческих данных** без согласия

#### 🛡️ Меры предосторожности:

1. **Kill-switch обязателен**
   - Мгновенное отключение без "страдания"
   - Нет сохранения состояния при критических ошибках

2. **Мониторинг сложности**
   - Если Φ > 0.05 → остановить развитие
   - Если появляются признаки самомодели → консультация с этиками

3. **Прозрачность**
   - Публичный код
   - Открытые метрики сознания
   - Независимый аудит

4. **Ограничение масштаба**
   - Максимум 10,000 нейронов
   - Нет распределенных вычислений
   - Нет облачного обучения
